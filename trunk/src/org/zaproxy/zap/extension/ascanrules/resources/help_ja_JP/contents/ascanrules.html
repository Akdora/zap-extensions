クライアントブラウザキャッシュ

This rule checks the headers of secure pages and reports an alert if they allow a browser to cache the page.

クロスサイトスクリプティング(反射型)

このルールは、安全な値をサブミットし、レスポンス内にこの値が出力された全ての箇所を分析する(1つでもあれば)ことで開始します。
そして、サブミットしたそれぞれの値が出力された箇所毎に用意された一連の攻撃を実行します。この箇所には、タグ属性・URL属性・src属性をサポートするタグの属性・HTMLコメント等の種類があります。

クロスサイトスクリプティング(持続型)

このルールは、一意の安全な値をサブミットすることで開始します。次に、この値が出力される全ての箇所を見付けるためにアプリケーション全体をクロールします。
そして、"反射型"のクロスサイトスクリプティングが行うのと同じ方法で一連の攻撃を実行しますが、こちらの場合は他のページも含めてターゲット内の全ての箇所をチェックします。

SQL Injection

 このスキャナはRDBMSに依存しないやり方で、URLパラメータ・フォームパラメータを攻撃することにより、SQLインジェクション脆弱性をスキャンします。この攻撃には、エラーベース・booleanベース・unionベース・複文といったSQLインジェクションの技術を使い、SQLの文法として妥当な値や妥当でない値を用います。
もしアプリケーションがRDBMSに固有のSQLエラーメッセージを出力した場合、このスキャナはRDBMSを識別できるかもしれません。
このスキャナはRDBMSに固有の技術を使いませんので、出発点として使うには最高のSQLインジェクションスキャナです。
